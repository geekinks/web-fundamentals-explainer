# Web Fundamentals Explainer Project

## Project Overview
This project explains how the web works from a systems perspective.
The goal is to demonstrate clear understanding of client–server architecture,
data flow, and the roles of core web technologies.

This is not a coding-heavy project.
It is a thinking and communication project.

---

## Q1 What Is the Web?

A website is a group of files kept together in one place. These files contain things like text, pictures, and designs (HTML and CSS), as well as actions and functions (JavaScript) that make the website work when you open it on the internet.
---

## Q2 Client–Server Architecture
Explain:
A client is any device like a mobile phone, laptop, or tablet that people use to visit a website or app. A server is the one that keeps the website’s files and information, and sends them to the client when it is requested.

## Q3 How Data Flows
## 1. When a user visits a website

-The user types a website address or clicks a link.

-The browser asks the server for the website.

-The server sends the website files (like text, images, and layout).

-The browser displays the website on the user’s screen.

## 2. When a user clicks a button

-The browser notices the button was clicked.

-The button may trigger an action, such as showing new information, opening another page, or sending a request to the server.

-The result of that action is then shown to the user.

## 3. When a user submits a form

-The user fills in details (like name or email) and clicks submit.

-The browser sends the entered information to the server.

-The server processes the information (for example, saving it).

-The server sends a response, such as a confirmation message or a new page, which the user sees.

## Q4 Roles of HTML, CSS, and JavaScript
Roles of HTML, CSS, and JavaScript (Simple Explanation)

HTML

Problem it solves: It gives a website its structure and content, like text, images, and buttons.

CSS

Problem it solves: It makes the website look good by controlling colors, fonts, spacing, and layout.

JavaScript

Problem it solves: It makes the website interactive, like clicking buttons, showing messages, or updating content without reloading the page.

## What happens when responsibilities are mixed

The website becomes hard to understand.

Fixing problems takes more time.

Changing design or features can break other parts of the website.

## Why separation matters

Each part is easier to manage.

The website is cleaner and more organized.

Designers and developers can work better together.

Updates and fixes become faster and safer.

## Q5 Common Beginner Mistakes
## 1. Skipping the basics

Why it happens:
Many beginners rush to build “big” projects or copy code without first understanding simple things like HTML structure or basic CSS.

## 2. Copying code without understanding it

Why it happens:
It feels faster to copy from tutorials or the internet instead of learning how the code actually works.

## 3. Not practicing enough

Why it happens:
Some beginners watch many tutorials but don’t build small projects on their own, so the knowledge doesn’t stick.
